galaxy_dive.utils.data_operations.align_axes(arr, new_z_axis, point=array([0., 0., 0.]), align_frame=True)[source]¶

Rotate a series of 3d coordinates s.t. the new z axis is aligned with the input vector new_z_axis

Parameters:
  • arr (ndarray) – The coordinates to be rotated. Dimensions must be (n_coords, 3).
  • new_z_axis (vector arr) – The vector along which the new z axis will be aligned.
  • point (vector arr) – The point to rotate around. If none, defaults to the origin.
  • align_frame (bool) – If True, the coordinates are aligned along new_z_axis, but the data keep their same coordinates as before. I.e. we rotate our perspective s.t. new_z_axis is the new z axis. If False, we rotate the data instead, and keep the coordinates the same. I.e. we rotate the data z-axis to be aligned with new_z_axis, but we still view the data from the original perspective. This just changes the sign of the angle of rotation.
Returns:

The rotated array.
Return type:

rotated_arr (ndarray)
